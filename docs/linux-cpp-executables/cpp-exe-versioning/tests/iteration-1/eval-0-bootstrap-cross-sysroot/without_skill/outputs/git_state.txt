    Add git-tag-based semantic versioning mechanism
    
    - Version is computed from annotated tags (vMAJOR.MINOR.PATCH) via
      git describe at build time; VERSION file is a tarball fallback.
    - version.h is generated into the build tree each build and embeds
      MAJOR/MINOR/PATCH, the full SemVer string and git describe output.
    - CMake project() version is driven by the same source of truth.
    - sensor-daemon now reports its version (--version).
